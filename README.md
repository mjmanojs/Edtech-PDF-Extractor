An advanced, dual-mode application that functions both as an automated PDF extraction tool for bypassing deeply restricted educational course viewers, and as an offline, lossless Bulk PDF Inverter that algorithmically creates authentic dark-mode textbook conversions heavily optimized for e-ink or night reading. 

**Note**: This suite requires **Node.js** installed locally on your machine to host the background automated server and run out of the browser. All processing happens 100% securely and natively on your device.

## Dual-Mode Features
### 1. EdTech PDF Extractor
- 📚 **Canvas Lazy-Loading Bypass**: Extracts virtualized pages continuously while scrolling through lessons.
- 🔓 **Raw API Hooking**: Directly interfaces with restricted `window.PDFViewerApplication` memory space.
- 💾 **Protected Download Bridge**: Bypasses browser blocking by piping massive output files directly to `Downloads/extracted pdf/`.

### 2. Lossless Bulk PDF Inverter
- 🌗 **Zero-Loss Inverse Processing**: Bypasses standard compression to render incredibly high resolution (150/300/450 PPI) inversions.
- 🚀 **Hardware-Accelerated Encoding**: Dynamically routes massive textbook arrays into native C++ Chrome JPEG integrations to eliminate pure-JS queue memory crashes.
- 🗃️ **Intelligent Queue Handling**: Features auto-retries, crash skipping, numeric metadata directory tracking, and dedicated routing directly to your local `Downloads/inverted pdf/`.

## 🚀 Quick Start & Installation

To run the unified dashboard locally entirely on your own secure ecosystem, use your system terminal or command prompt (like PowerShell or Command Prompt) and run the following execution blocks:

### 1. Clone & Setup
```bash
git clone https://github.com/mjmanojs/Edtech-PDF-Extractor.git
cd EdTech-PDF-Extractor

# Install all necessary puppeteer and node backends
npm install
```

### 2. Launch the Application Server
*(We strongly recommend allocating maximum memory space to Node due to the extreme rendering resolutions provided by the Inverter tool)*

```bash
# Safely start the server with an 8GB memory allowance
node --max-old-space-size=8192 server.js
```

### 3. Open the Dashboard
Once the terminal states the server is running, simply open an internet browser (like Chrome, Edge, or Brave) and navigate to:
**[http://localhost:3000](http://localhost:3000)**

*(The interface inherently features persistent state mapping, returning you safely to whichever module you were utilizing previously even across tabs!)*
